Overview

NSHA (Null SHA Fixer) is a command-line tool to detect and fix null SHA (0000000...) and broken tree object issues in Git repositories.

What is a Null SHA Issue?

A null SHA issue occurs when Git references (branches, tags, or commits) point to invalid or non-existent objects, typically represented as 0000000000000000000000000000000000000000. These issues can cause:

  • Repository corruption and data loss
  • Failed git operations (push, pull, clone)
  • Broken commit history
  • Missing branches or tags
  • Inability to access certain parts of the repository

Key Capabilities

Diagnose

Comprehensive repository scanning to detect null SHA references, missing commits, broken parents, and corrupted tree objects.

Fix

Automated repair with history rewriting when needed. Includes dry-run mode to preview changes before applying.

Verify

Post-fix verification to ensure repository integrity and confirm all issues have been resolved.

Backup

Automatic repository backup before any modifications, with integrity verification and rollback capability.

Core Features

  • Fast Performance: Optimized for maximum speed and efficiency
  • Cross-Platform: Native support for Windows, Linux, and macOS (Intel and ARM)
  • Safe Operations: Automatic backups and dry-run mode prevent data loss
  • Comprehensive Logging: Detailed logs and reports saved to user's home directory
  • Beautiful CLI: Colored output with progress indicators for better user experience
  • Multiple Issue Types: Detects and fixes null SHA, missing commits, broken parents, and corrupted trees
  • History Rewriting: Rewrites repository history to fix corrupted commits and trees
  • Garbage Collection: Automatic cleanup of orphaned objects after fixes

When to Use NSHA

NSHA is designed for situations where your Git repository has become corrupted or contains invalid references. Common scenarios include:

  • After a failed git operation or interrupted process
  • When git fsck reports errors
  • After recovering from disk corruption or system crashes
  • When branches or tags point to non-existent commits
  • After manual repository manipulation or migration
  • When encountering "bad object" or "missing object" errors

Usage

Basic Commands

NSHA provides three main commands for managing repository issues:

1. Diagnose Issues

Check if your repository has any null SHA or broken tree issues:

# Diagnose current directory
nsha diagnose

# Diagnose specific repository
nsha diagnose --repo /path/to/repo

# Verbose output
nsha diagnose --verbose

Example Output:

[STEP 1] Diagnosing repository...
[WARNING] Found 3 issue(s):
  1. [null-sha] refs/tags/null-tag: Reference has null SHA
  2. [null-sha] refs/heads/broken-branch: Reference has null SHA
  3. [missing-commit] 0000000000000000000000000000000000000001: Cannot read commit

2. Fix Issues

Automatically fix all detected issues:

# Dry-run (preview changes without applying)
nsha fix --dry-run

# Fix with confirmation prompt
nsha fix

# Fix without confirmation
nsha fix --yes

# Fix with verbose output
nsha fix --verbose

# Fix specific repository
nsha fix --repo /path/to/repo

Example Output:

[STEP 1] Diagnosing repository...
[INFO] Logging to: C:\Users\username\nsha\20240203-105946

[STEP 2] Creating repository backup...
[SUCCESS] Backup created successfully
[SUCCESS] Directory copy backup verified successfully

[STEP 3] Fixing null SHA issues...
[SUCCESS] Fixed 2 issue(s)!

3. Verify Repository

Verify that all issues have been resolved:

# Verify current directory
nsha verify

# Verify specific repository
nsha verify --repo /path/to/repo

# Verbose output
nsha verify --verbose

Example Output:

[STEP 1] Verifying repository integrity...
[SUCCESS] Repository is healthy! No issues found.

Command Flags

All commands support the following flags:

Flag Description Default
-r, --repo <path> Path to Git repository Current directory
-v, --verbose Enable verbose output with detailed information false
-h, --help Show help for the command -

Fix command additional flags:

Flag Description Default
--dry-run Preview changes without applying them false
-y, --yes Skip confirmation prompt false
-f, --force Force operation even with warnings false

Complete Workflow Example

Here's a complete workflow for diagnosing and fixing repository issues:

# Step 1: Navigate to your repository
cd /path/to/your/repository

# Step 2: Check for issues
nsha diagnose --verbose

# Step 3: Preview fixes (dry-run)
nsha fix --dry-run --verbose

# Step 4: Apply fixes
nsha fix --yes --verbose

# Step 5: Verify the fix
nsha verify

# Step 6: Push to remote (if needed)
git push origin --force --all
git push origin --force --tags

Architecture & Technical Specifications

How NSHA Works

NSHA follows a systematic, multi-step approach to detect and fix null SHA issues in Git repositories:

1

Diagnosis

Runs a comprehensive repository scan (similar to git fsck) to identify:

  • Null SHA references in branches, tags, and packed-refs
  • Missing commits and broken parent references
  • Corrupted tree objects
  • Dangling references
2

Backup Creation

Creates a complete backup of the repository before any modifications:

  • Attempts git bundle first (preserves full history)
  • Falls back to directory copy if bundle fails
  • Verifies backup integrity
  • Stores backup in user's home directory with timestamp
3

Cleanup Packed-Refs

Scans and cleans the packed-refs file:

  • Removes null SHA entries (0000000..., 0000000...1, etc.)
  • Cleans up duplicate references
  • Ensures clean state before fixes
4

Fix Null SHA Issues

Applies fixes using git plumbing commands:

  • Points null SHA references to valid commits
  • Updates or removes tags with null SHA
  • Fixes references to non-existent commits
  • Detects and reports corrupted trees
5

History Rewriting

Rewrites corrupted repository history:

  • Walks the entire commit graph
  • Rewrites commits with corrected parents and trees
  • Updates all branches and tags
  • Preserves commit metadata (author, date, message)
6

Garbage Collection

Cleans up the repository:

  • Runs git gc --prune=now --aggressive
  • Runs git prune --expire=now
  • Removes orphaned objects
  • Compacts repository
7

Verification

Final integrity check:

  • Runs git fsck to verify repository health
  • Confirms all issues are resolved
  • Reports any remaining issues

Issue Detection Types

NSHA can detect and fix the following types of repository issues with detailed real-time logging:

1. Null SHA References

Description: References (branches, tags, or packed-refs) pointing to the null SHA 0000000000000000000000000000000000000000 or similar invalid patterns.

Real-time Log Example:

[STEP 1] Diagnosing repository...
[WARNING] Found 2 issue(s):
  1. [null-sha] refs/tags/v1.0.0: Reference has null SHA
     SHA: 0000000000000000000000000000000000000000
  2. [null-sha] refs/heads/feature-branch: Reference has null SHA
     SHA: 0000000000000000000000000000000000000001

[STEP 3] Fixing null SHA issues...
[INFO] Fixing reference: refs/tags/v1.0.0
[INFO] Pointing to latest commit: a1b2c3d4e5f6...
[SUCCESS] Fixed refs/tags/v1.0.0

Fix Strategy: Point reference to a valid commit (usually HEAD or latest commit on the branch) or remove the reference if no valid target exists.

2. Missing Commits

Description: References pointing to commit objects that don't exist in the repository's object database.

Real-time Log Example:

[STEP 1] Diagnosing repository...
[WARNING] Found 1 issue(s):
  1. [missing-commit] refs/heads/old-branch: Cannot read commit
     SHA: abc123def456789012345678901234567890abcd
     Error: object not found

[STEP 3] Fixing null SHA issues...
[INFO] Fixing reference: refs/heads/old-branch
[INFO] Commit abc123d... not found
[INFO] Updating to parent commit: def456e...
[SUCCESS] Fixed refs/heads/old-branch

Fix Strategy: Update reference to point to the parent commit, or to HEAD if parent is also missing. If no valid commit exists, the reference is removed.

3. Broken Parents

Description: Commits that reference parent commits with null SHA or non-existent parent objects, breaking the commit history chain.

Real-time Log Example:

[STEP 1] Diagnosing repository...
[WARNING] Found 1 issue(s):
  1. [broken-parent] commit 789abc123def456: Has null SHA parent
     Commit: 789abc123def456789012345678901234567890
     Parent: 0000000000000000000000000000000000000000

[STEP 5] Rewriting history...
[INFO] Processing commit: 789abc1...
[INFO] Removing null SHA parent reference
[INFO] Creating new commit with fixed parents
[SUCCESS] Rewrote commit 789abc1... → 456def7...

Fix Strategy: Rewrite the commit history, removing null SHA parents or replacing them with valid parent commits. This preserves the commit message, author, and date.

4. Missing Trees

Description: Commits that reference tree objects (directory structures) that don't exist in the repository.

Real-time Log Example:

[STEP 1] Diagnosing repository...
[WARNING] Found 1 issue(s):
  1. [missing-tree] commit 456def789abc123: Cannot read tree
     Commit: 456def789abc123456789012345678901234567
     Tree: 111222333444555666777888999000aaabbbccc
     Error: tree object not found

[STEP 5] Rewriting history...
[INFO] Processing commit: 456def7...
[INFO] Tree 111222... not found
[INFO] Creating empty tree as replacement
[INFO] New tree SHA: dddeeefff000111222333444555666777888
[SUCCESS] Rewrote commit with new tree

Fix Strategy: Create a replacement tree (usually empty or copied from parent commit) and rewrite the commit to reference the new tree.

5. Corrupted Trees

Description: Tree objects that contain entries with null SHA or invalid object references, causing directory structure corruption.

Real-time Log Example:

[STEP 1] Diagnosing repository...
[WARNING] Found 1 issue(s):
  1. [corrupted-tree] tree 999888777666555: Contains null SHA entry
     Tree: 999888777666555444333222111000fffeeedd
     Entry: src/main.go → 0000000000000000000000000000000000000000

[STEP 5] Rewriting history...
[INFO] Processing tree: 9998887...
[INFO] Found corrupted entry: src/main.go
[INFO] Removing null SHA entry from tree
[INFO] Rebuilding tree without corrupted entries
[INFO] New tree SHA: aaabbbccc111222333444555666777888999
[SUCCESS] Rebuilt tree 9998887... → aaabbbc...

Fix Strategy: Rebuild the tree object, removing entries with null SHA or replacing them with valid objects from previous commits.

Troubleshooting

Common Issues and Solutions

"Repository not found" Error

Problem: NSHA cannot find the Git repository.

Solution:

# Make sure you're in a Git repository
cd /path/to/your/repository

# Or specify the repository path explicitly
nsha diagnose --repo /path/to/repository

"Permission Denied" Error

Problem: NSHA doesn't have permission to access the repository or create backups.

Solution:

  • Windows: Run PowerShell as Administrator
  • Linux/macOS: Use sudo or change repository permissions
# Linux/macOS: Run with sudo
sudo nsha fix --yes

# Or change permissions
chmod -R u+w /path/to/repository

"Backup Failed" Error

Problem: NSHA cannot create a backup of the repository.

Solution:

  • Check available disk space in your home directory
  • Ensure you have write permissions to ~/nsha/
  • Try manually creating the directory: mkdir -p ~/nsha

"Fix Failed" - Issues Still Remain

Problem: After running fix, some issues still exist.

Solution:

# Run fix multiple times if needed
nsha fix --yes --verbose

# Check the logs for details
cat ~/nsha/YYYYMMDD-HHMMSS/nsha.log

# Verify the repository
nsha verify --verbose

"Cannot Push to Remote" After Fix

Problem: Git refuses to push after NSHA rewrites history.

Solution:

History rewriting changes commit SHAs, requiring force push:

# Force push all branches
git push origin --force --all

# Force push all tags
git push origin --force --tags

# Warning: Coordinate with team members before force pushing!

Windows SmartScreen Warning

Problem: Windows shows "unsafe" warning when downloading .exe files.

Solution:

This is normal for unsigned executables. To proceed:

  1. Click "More info" or "Keep anyway"
  2. Right-click the file → Properties → Check "Unblock" → OK
  3. Run the file

The warning will disappear as more users download the tool.

Getting Help

If you encounter issues not covered here:

  • Check the logs: Located in ~/nsha/YYYYMMDD-HHMMSS/nsha.log
  • Run with verbose mode: Add --verbose flag to any command
  • Use the comments section: Scroll down to ask questions

Best Practices

  • Always use dry-run first: Preview changes with nsha fix --dry-run
  • Keep backups: NSHA creates automatic backups, but keep your own as well
  • Coordinate with team: Inform team members before force pushing
  • Test locally first: Fix issues on a local clone before pushing
  • Review logs: Check logs after fixes to understand what changed
  • Verify after fix: Always run nsha verify after fixing
